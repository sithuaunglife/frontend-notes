# Postman

## Facts
- Postman is an API testing tool.
- To send (POST) data in Postman, you need to write the data in the Body, select raw, and choose JSON.
- In Postman, if multiple endpoints share the same base URL, select the parent collection and define a baseUrl variable there to centralize it (e.g. http://localhost:8000).
- Remember to save in Postman after creating or editing variables, unless autosave is turned on.
- The UI sends a request, and the terminal responds.
- Fetching all data using a ```GET``` request is called the index action.
- Fetching a single item using a ```GET``` request is called the show action.
- Sending data using a ```POST``` request is called the store action.
- Updating data using a ```PUT``` request is called the update action.
- Updating a single item partially using a ```PATCH``` request is handled by the update action. Example: update-price
- Deleting items using a ```DELETE``` request is called the destroy action.
- When creating data with a ```POST``` request, an id is not required because it is generated by the server.
- Postman can generate code snippets for API requests.
- If someone runs the API on their own machine, they can use localhost (127.0.0.1). localhost only refers to the current computer. If you are using an API that runs on someone else’s server, you cannot use localhost. You must use the real server domain or IP address.
